Languages

What should you learn first?

A programming language is a tool built for a specific purpose. For most beginners, HTML is the right place to start. Below is what five of the most common beginner languages look like, each doing the same thing in a different way.

HTML structures the content of every webpage. It's not technically a programming language, but it is where every web developer begins.

<!-- A comment. The browser ignores this. -->
<h1>Hello, World!</h1>
<p>This is a paragraph of text.</p>
<a href="https://example.com">This is a link</a>
<img src="photo.jpg" alt="A description">
Start here if you want to build websites and have never written a line of code. You will see results in minutes.

CSS controls how HTML looks: colors, fonts, spacing, layout. It's what separates a raw document from a designed page.

/* Change how the whole page looks */
body {
  background-color: #faf7f2;
  font-family:      Georgia, serif;
  color:            #1a1612;
  max-width:        700px;
  margin:           0 auto;
}
h1 { font-size: 2.5rem; }
Start here if you are drawn to design. CSS is immediately gratifying: one line changes the entire appearance of a page.

JavaScript makes websites interactive. Buttons that respond, forms that validate, animations, games — all of it is JavaScript.

// Store a value in a variable
let name = "World";
// Print to the browser's console
console.log(`Hello, ${name}!`);
// React when a button is clicked
document.querySelector("button")
  .addEventListener("click", () => {
    alert("You clicked it!");
  });
Start here if you already know basic HTML and CSS and want to add real interactivity to your pages.

Python reads almost like plain English. It is the dominant language for data science, AI, and automation, and one of the most beginner-friendly options available.

# Print a message to the terminal
print("Hello, World!")
# Define a reusable function
def greet(name):
    return f"Hello, {name}!"
print(greet("World"))
Start here if you are interested in data, AI, science, or automation rather than building websites.

Swift is Apple's language for building iPhone, iPad, and Mac apps. It is modern, fast, and designed from the ground up to be readable.

// Swift — used for iOS and macOS apps
import Foundation
let greeting: String = "Hello, World!"
print(greeting)
func greet(name: String) -> String {
    return "Hello, \(name)!"
}
Start here if you own a Mac and want to build iPhone or iPad apps. You will need Xcode, which is also free.

Learning roadmap

A clear path from zero to building things.

The most common mistake beginners make is trying to learn everything at once. Follow this sequence. Each step builds naturally on the one before it.

1
Weeks 1–2  ·  ~8 hours total

HTML Fundamentals

Learn what tags are and how web pages are structured. Build your first page. Headings, paragraphs, links, images, lists. Small scope, high confidence.

2
Weeks 2–4  ·  ~12 hours total

CSS Fundamentals

Style your HTML. Colors, fonts, padding, borders. Then flexbox for layouts. This is where things start looking like real websites.

3
Month 2  ·  ~20 hours total

JavaScript Basics

Variables, functions, conditionals, loops. Then DOM manipulation: making buttons respond, updating text, handling user input. This is the turning point.

4
Month 2–3  ·  ~6 hours total

Git and GitHub

Save your work. Undo mistakes. Share your projects publicly. Host websites for free. This is the skill that makes everything you build visible to the world.

5
Month 3 onward

Build Projects

This is where real learning happens. Pick something small. Build it. Show it to someone. Get feedback. Pick something slightly harder. Repeat indefinitely.

How this page works

How the dark and light mode on this page was built.

The toggle in the navigation is not a library or a plugin. It is roughly twenty lines of CSS and one small JavaScript function. Here is exactly how it works, so you can add it to any site you build.

1

Define colors as CSS variables

Instead of writing color values like #faf7f2 directly into every rule, you define named variables once inside :root. Every element that references var(--bg) will automatically update when that variable's value changes. 

:root {
  --bg:     #faf7f2;
  --text:   #1a1612;
  --accent: #b85c2a;
}
body {
  background: var(--bg);
  color:      var(--text);
}
2

Override the variables for dark mode

When the <html> element has the attribute data-theme="dark", CSS redefines those same variable names with dark values. No element rules change at all — only what the variables hold. 

[data-theme="dark"] {
  --bg:     #100e0b;
  --text:   #ede6da;
  --accent: #d4784a;
}
3

Add smooth transitions

Adding a CSS transition to body means the color change animates smoothly rather than snapping. One line handles everything. 

body {
  transition: background 0.3s ease, color 0.3s ease;
}
4

Toggle with JavaScript, remember the preference

One small function switches the attribute on and off. localStorage saves the choice so it persists when the user returns to the page. 

function toggleTheme() {
  const html   = document.documentElement;
  const isDark = html.dataset.theme === "dark";
  html.dataset.theme = isDark ? "light" : "dark";
  localStorage.setItem("theme", html.dataset.theme);
}

// Restore saved preference on page load
const saved = localStorage.getItem("theme");
if (saved) document.documentElement.dataset.theme = saved;

Mindset

The part no one talks about.

Technical skills are learnable by anyone. The real variable is what you tell yourself when it gets difficult. Here is what actually matters.

Everyone who codes well today was once completely lost. The only difference between them and someone who gave up is that they kept going anyway. The shared experience of every working developer
Bugs are normal.

Senior engineers spend most of their time debugging. Getting stuck is not failure. It is the work itself.

Google everything.

Real developers search constantly. Knowing how to find answers is more valuable than memorizing syntax.

Build over watch.

Tutorials feel productive. Building things is what actually moves you forward. Even broken, incomplete projects teach more.

Consistency wins.

Thirty minutes daily beats four-hour weekend sessions. Frequency matters more than duration when learning anything new.

Publishing your work

How to put a page online, for free, right now.

GitHub Pages hosts static websites at no cost. Follow these five steps and you will have a real URL to share within a few minutes.

1

Create a GitHub account

Go to github.com and sign up. Choose your username thoughtfully — it will appear in your site's URL.

2

Create a new repository

Click the + icon at the top right, then "New repository." Name it coding-guide. Check "Add a README." Click "Create repository."

3

Upload your index.html

Inside the repository, click "Add file" then "Upload files." Drag your index.html into the window. Scroll down and click "Commit changes."

4

Enable GitHub Pages

Go to the Settings tab of your repository. Find "Pages" in the left sidebar. Under "Branch," select main and click Save.

5

Your site is live

Within about sixty seconds your site will be at https://yourusername.github.io/coding-guide. That is a real URL. Share it.
