Languages

What should you learn first?

A programming language is a tool built for a specific purpose. For most beginners, HTML is the right place to start. Below is what five of the most common beginner languages look like, each doing the same thing in a different way.

HTML structures the content of every webpage. It is not technically a programming language, but it is where every web developer begins.

<!-- A comment. The browser ignores this. -->
<h1>Hello, World!</h1>
<p>This is a paragraph of text.</p>
<a href="https://example.com">This is a link</a>
<img src="photo.jpg" alt="A description">
Start here if you want to build websites and have never written a line of code. You will see results in minutes.

CSS controls how HTML looks: colors, fonts, spacing, layout. It is what separates a raw document from a designed page.

/* Change how the whole page looks */
body {
  background-color: #faf7f2;
  font-family:      Georgia, serif;
  color:            #1a1612;
  max-width:        700px;
  margin:           0 auto;
}
h1 {
  font-size:        2.5rem;
}
Start here if you are drawn to design. CSS is immediately gratifying: one line changes the entire appearance of a page.

JavaScript makes websites interactive. Buttons that respond, forms that validate, animations, games — all of it is JavaScript.

// Store a value in a variable
let name = "World";

// Print to the browser's console
console.log(`Hello, ${name}!`);

// React when a button is clicked
document.querySelector("button")
  .addEventListener("click", () => {
    alert("You clicked it!");
  });
Start here if you already know basic HTML and CSS and want to add real interactivity to your pages.

Python reads almost like plain English. It is the dominant language for data science, AI, and automation, and one of the most beginner-friendly options available.

# Print a message to the terminal
print("Hello, World!")

# Define a reusable function
def greet(name):
    return f"Hello, {name}!"

print(greet("World"))
Start here if you are interested in data, AI, science, or automation rather than building websites.

Swift is Apple's language for building iPhone, iPad, and Mac apps. It is modern, fast, and designed from the ground up to be readable.

// Swift — used for iOS and macOS apps
import Foundation

let greeting: String = "Hello, World!"
print(greeting)

func greet(name: String) -> String {
    return "Hello, \(name)!"
}
Start here if you own a Mac and want to build iPhone or iPad apps. You will need Xcode, which is also free.

The Roadmap

Your first six months.

Learning to code is a marathon, not a sprint. Follow this path to go from zero to building your first real applications.

1
Month 1

The Foundations (HTML & CSS)

Learn how to structure content and make it look beautiful. Build static pages like a personal resume or a landing page for a fictional product.

2
Month 2-3

Logic & Interactivity (JavaScript)

Learn how to make your pages do things. Variables, functions, and manipulating the Document Object Model (DOM). Build a calculator or a to-do list.

3
Month 4

Tooling & Workflow (Git & Terminal)

Learn how professional developers work. Version control with Git, using the command line, and managing packages. This is where you become efficient.

4
Month 5-6

Building for Real (Projects)

Combine everything you've learned. Build a weather app using real data, or a small social network profile. Focus on finishing and deploying your work.

How it works

Under the hood.

When you type a URL into your browser, a complex series of events happens in milliseconds. Here is the simplified version.

01

The Request

Your browser sends a message across the internet to a server (a powerful computer) asking for the files for that website.

02

The Response

The server finds the HTML, CSS, and JavaScript files and sends them back to your computer in small pieces called packets.

03

The Rendering

Your browser's "engine" reads the code and turns it into the colors, images, and text you see on your screen.

The Mindset

Thinking like a developer.

Coding is more about problem-solving than it is about memorizing syntax. You need to change how you approach challenges.

"The most important skill of a computer scientist is problem solving. It means the ability to formulate problems, think creatively about solutions, and express a solution clearly and accurately." — Allen B. Downey

Break it down

Never try to solve a big problem all at once. Break it into tiny, manageable steps. If a step feels hard, it’s still too big.

Embrace the error

Errors aren't failures; they are the most valuable feedback you get. Read them carefully—they usually tell you exactly what’s wrong.

Google is your friend

Professional developers google things constantly. Knowing what to search for is more important than knowing everything by heart.

The final step

Go live on GitHub.

Once you've built your website, it's time to share it with the world. GitHub Pages makes this incredibly easy and free.

1

Create a Repository

Log in to GitHub and click "New" to create a repository. Give it a name like "my-first-website."

2

Upload your files

Drag and drop your index.html (and any CSS or image files) directly into the browser to upload them.

3

Enable Pages

Go to Settings > Pages. Select the "main" branch and click "Save."

4

Share the link

GitHub will provide a URL (like yourname.github.io/my-first-website). Your site is now live for anyone to see!
